15  まとめと評価

本実験の評価では、完成したプログラム、レポート、口頭説明を総合して判断する。 ここでは提出物・レポートの観点・口頭説明の観点を整理する。

15.1 提出物

提出物は次の通りである。

  • reversi_yourid.py：本書の例題・練習で作成したすべての関数を含む単一の Python ファイル。
  • test_reversi.py：自分が書いた単体テスト。
  • 最終レポート：実験条件、結果、考察を記したもの。
  • 対戦ログ・実験結果：レポートに引用したログファイル。
  • （任意）独自に作成したスクリプトやノートブック。

reversi_yourid.py には、

  • 第6章で実装した盤面のルール部分。
  • 第7章で実装した minimax 法。
  • 第8章で実装した αβ 法。
  • 第9～10章で実装した評価関数。

を含めること。 独自に追加した関数には、何を評価しているのか、どのような意図で使うのかが分かるコメントまたはdocstringを付ける。

15.2 レポートの観点

レポートでは次の内容を説明する。

  • オセロプログラム全体の構成（モジュール、関数の依存関係）。
  • 合法手判定と石の反転の実装方法。
  • minimax 法の実装方法。
  • αβ 枝刈りの実装方法と minimax 法との節点数の比較。
  • 評価関数の設計意図（各 board_evalN を何のために入れているか、重みをどう決めたか）。
  • 改良前後の実験結果（条件の表、対戦結果、勝敗）。
  • テストや Git をどのように使ったか（pytest の出力、git log --oneline の貼り付け）。
  • 考察（うまくいったこと、うまくいかなかったこと、次にやるとしたら何をするか）。

レポートのフォーマットは Markdown / PDF / Word のいずれでも構わないが、コードブロックと図表を読みやすく配置すること。

15.3 口頭説明

口頭説明（プレゼンテーションや個別質疑）では、主に次の点を確認する。

  • 自分のプログラムの主要関数をコードを見ながら説明できるか。
  • 評価関数の工夫を、なぜそうしたかまで説明できるか。
  • 実験結果に基づいて、何が良くなり、何が課題として残ったかを説明できるか。
  • エラーや不具合に対して、どのように調査し、どのように直したかを説明できるか。

評価では、勝敗だけでなく、

  • 実装の正しさ
  • 説明の明確さ
  • 開発過程の記録（Git ログ、テスト）
  • 実験計画と考察の妥当性

を重視する。 派手な手法を採用していなくても、自分が考えて改良し、実験で確かめたことが伝われば十分に高評価となる。

15.4 評価の重み（目安）

観点 重み
基本機能の完成度（第6～8章） 40%
評価関数の設計と実験（第9～10章） 25%
Git・テストの活用（第3～4章） 10%
レポート・口頭説明の質 20%
発展課題（第11～12章のいずれか）への取り組み +5%（加点）

第11章（メタヒューリスティクスによる最適化）または第12章（機械学習）への取り組みは加点対象である。 着手のみでも記録があれば評価する。 最終的にベースラインに勝てなくても、何を試し、結果がどうだったかを記録できていれば評価する。

15.5 学びの振り返り

最後に、本実験全体を通じて意識してほしい点をまとめる。

  • 大きなシステムは分割して作り、テストで動作を保証しながら積み上げる。
  • うまくいった状態を git commit で残しておくと、失敗からの復帰が容易になる。
  • 評価関数の改良は実験で確かめる。一度の対戦で結論を出さない。
  • 強い AI を作ることそのものよりも、何を試し、どう判断したかが知能工学実験の中心である。
