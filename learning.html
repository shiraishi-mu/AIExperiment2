13  発展：機械学習による評価関数の構築

第11章では、評価関数の重みをメタヒューリスティクスで最適化した。 本章ではさらに踏み込み、評価関数そのものをデータから学習するアプローチを紹介する。 これは現代の AI 研究の中心となっている機械学習（machine learning） の入門にもなる。

Note第11章との違い
章 探索する対象 入力 出力
第11章 メタヒューリスティクス 既知の評価関数の重み \(\mathbf{w}\) 対戦の勝率 最適な \(\mathbf{w}\)
第12章 機械学習 評価関数の関数形そのもの（モデルパラメータ） 局面と勝敗のデータ 局面 → 評価値の写像

メタヒューリスティクスは「人間が組んだ式の係数」を最適化するのに対し、機械学習は「式の形」もある程度データから決める。

本章は発展課題である。 本実験の必修範囲を完成させ、pytest で動作を確認してから取り組むこと。

13.1 本章の到達目標

  • 自己対戦で学習用データセットを作成できる。
  • 線形回帰またはロジスティック回帰で局面 → 勝敗の関係を学習できる。
  • 学習した重みを eval_node() に組み込み、対戦で性能を確認できる。

13.2 学習用データの準備

13.2.1 自己対戦によるデータ収集

自己対戦の各局面に対し、その局面の特徴量と最終的な勝敗を組にして記録する。

import csv

def collect_self_play(n_games, depth, output_path):
    """自己対戦を行い、(features, winner) のペアを CSV に保存する。"""
    rows = []
    for g in range(n_games):
        board = initial_board()
        turn = BOARD_SENTE
        count = 1
        history = []   # この対戦中の (features, turn) の系列

        while True:
            state = board_state(board, turn)
            if isinstance(state, float):
                break
            if state == "pass":
                turn = change_turn(turn)
                continue

            features = extract_features(board)   # 後述
            history.append(features)

            n = Node(board, turn, 0, 0, 0.0)
            best = alpha_beta(n, turn, count, depth, None, None)
            board_move(board, best.row, best.col, turn)
            turn = change_turn(turn)
            count += 1

        # 勝者を全局面のラベルとして付与
        sente = board_number_check(board, BOARD_SENTE)
        gote = board_number_check(board, BOARD_GOTE)
        if sente > gote:
            label = 1
        elif sente < gote:
            label = -1
        else:
            label = 0

        for feat in history:
            rows.append(feat + [label])

    with open(output_path, "w", newline="") as f:
        csv.writer(f).writerows(rows)

13.2.2 練習44：特徴量の設計

局面 board から数値ベクトルを抽出する関数 extract_features(board) を定義せよ。 最初は素朴に、第9章で定義した評価項目をそのまま並べるとよい。

def extract_features(board):
    return [
        board_eval(board),
        board_eval2(board, 1),
        board_eval3(board, BOARD_SENTE, 1) - board_eval3(board, BOARD_GOTE, 1),
        board_eval4(board, 1),
        board_eval5(board, BOARD_SENTE, 1) + board_eval5(board, BOARD_GOTE, 1),
    ]

ここで返るリストの長さが特徴量の次元になる。 発展課題として、自分が新たに思いついた特徴量（たとえば「角の所有数の差」「辺の安定石の数」など）を追加して次元を増やせる。

13.3 線形回帰による評価関数

最も単純なモデルは線形回帰である。 特徴量ベクトル \(\mathbf{x}\)、重みベクトル \(\boldsymbol{\beta}\)、バイアス \(b\) について

\[ \hat{y} = \boldsymbol{\beta}^\top \mathbf{x} + b \]

を「先手の勝ち \(1\)、後手の勝ち \(-1\)、引き分け \(0\)」のラベルにフィットさせる。

13.3.1 練習45：scikit-learn で学習する

import numpy as np
from sklearn.linear_model import LinearRegression

data = np.loadtxt("self_play.csv", delimiter=",")
X = data[:, :-1]
y = data[:, -1]

model = LinearRegression()
model.fit(X, y)
print("coefficients:", model.coef_)
print("intercept:   ", model.intercept_)

学習された係数 model.coef_ は、特徴量の各次元に対する重みである。

13.3.2 練習46：学習結果を eval_node() に組み込む

class LearnedEvaluator:
    def __init__(self, coef, intercept):
        self.coef = coef
        self.intercept = intercept

    def __call__(self, node, turn, count):
        x = extract_features(node.board)
        return float(np.dot(self.coef, x) + self.intercept)


# 使用例
learned = LearnedEvaluator(model.coef_, model.intercept_)
set_eval_node(learned)   # 第11章で導入した差し替え機構を使う

第11章で導入した set_eval_node 機構をそのまま流用できる。

13.3.3 練習47：手動評価関数との対戦

学習した評価関数と、第9章の手動評価関数 eval_node を対戦させ、勝率を比較せよ。 データ量・特徴量・モデルの違いで結果がどう変わるかを観察する。

13.4 ロジスティック回帰

線形回帰は連続値を予測するモデルだが、勝ち負けは本来分類の問題である。 ロジスティック回帰を使うと、予測値を確率として解釈できる。

from sklearn.linear_model import LogisticRegression

# 引き分けは除き、勝ち(1) と 負け(-1) の二値に絞る
mask = y != 0
X2, y2 = X[mask], (y[mask] > 0).astype(int)

model = LogisticRegression(max_iter=1000)
model.fit(X2, y2)

predict_proba で得られる「先手勝率」をそのまま評価値として使える。

13.5 小規模ニューラルネットワーク

特徴量に非線形な関係があると思える場合は、多層パーセプトロン（MLP） を使うとよい。

from sklearn.neural_network import MLPRegressor

model = MLPRegressor(hidden_layer_sizes=(16, 8), max_iter=2000, random_state=0)
model.fit(X, y)

PyTorch を使うとさらに柔軟なモデル設計ができるが、本実験の範囲では scikit-learn の MLP で十分である。

13.6 学習データの注意点

13.6.1 データの偏り

自己対戦データは、今の AI の打ち手分布に強く依存している。 そのため、学習で得られる評価関数も「現在の AI が好む局面に最適化された」ものになる。 本来未知の局面（たとえば人間との対戦で出てくる局面）には弱いままであることがある。

これを和らげる方法として、

  • 異なる重みの AI を複数用意し、それらの組み合わせで自己対戦する。
  • 序盤の数手をランダムにすることで、棋譜の多様性を増やす。
  • 公開棋譜（オセロのプロ棋譜など）を学習に混ぜる。

などが考えられる。

13.6.2 過学習

特徴量の次元やモデルが大きすぎると、学習データの「ノイズ」まで覚えてしまい未知データで弱くなる。 これを過学習（overfitting） と呼ぶ。 対策としては、

  • 学習データ・検証データの分割。
  • 正則化（Ridge、Lasso、LogisticRegression(C=...) など）。
  • モデルの規模を小さくする。

がある。本実験では学習データを小さく取らざるを得ないので、まずは少ない特徴量から始めるのが安全である。

13.7 発展：強化学習

オセロは状態と行動が明確に定義されているため、強化学習の題材としても扱いやすい。

  • TD 学習：「現在の局面の評価」と「1 手進めた後の局面の評価」が一致するように、評価関数を逐次更新する。
  • AlphaZero 流のセルフプレイ：方策ネットと価値ネットを同時に学習し、モンテカルロ木探索と組み合わせる。

これらは本実験の必修範囲を大きく超える。 興味のある学生は卒業研究や個人プロジェクトとして取り組むことを推奨する。

13.8 レポートに含める内容（学習を選択した場合）

  • 使用した特徴量の説明
  • データ収集の方法（自己対戦の重み、対戦数、深さ）
  • 用いたモデル（線形回帰／ロジスティック回帰／MLP など）
  • 学習結果（係数、損失、検証スコア）
  • 学習した評価関数 vs 手動評価関数の対戦結果
  • 過学習・データの偏りなどに対して行った対策
  • 第11章のメタヒューリスティクス的アプローチとの比較

13.9 まとめ

  • 評価関数をデータから学習する場合、自己対戦で局面と勝敗のデータを集め、線形回帰や MLP で勝敗を予測する形に持ち込む。
  • メタヒューリスティクスとは異なり、特徴量の設計と過学習・データの偏りへの気配りが必要になる。
  • 強化学習はさらに高度だが、本実験の枠組み（盤面表現・合法手判定・木探索）はそのまま土台として使える。
