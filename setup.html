3  環境構築と準備

本実験で用いるのは、これまでの講義・演習でも扱ってきたPythonである。 ただしオンラインサービスである Google Colaboratory ではなく、ローカル環境にPythonをインストールして実験を行う。 2026年度は、知能工学実験室に備え付けられているデスクトップPCを用いて実験を進める。

本学では BYOD（Bring Your Own Device）を推進しており、学生は個人のノートPCを持ち学習を進めている。 しかし学生ごとにOSや環境が異なるため、本実験のように環境依存性の強い実験では、共通化が必要になる。 そこで、統一された環境を提供するためVSCode + Dev Containers + Docker による仮想環境を導入する。

手順の概要は次の通りである。

  1. VSCode をインストールする。
  2. VSCode に Dev Containers 拡張機能をインストールする。
  3. Docker をインストールする。
  4. 配布される lab_env.zip を展開して VSCode で開き、Dev Containers でコンテナを起動する。
  5. コンテナ内のターミナルで python --version を確認する。

3.1 Visual Studio Code

Visual Studio Code（VSCode）はMicrosoftが提供する無料のコードエディタである。 本実験では Python の編集、実行、デバッグ、テスト、Git 操作を VSCode 上で完結させる。

3.1.1 VSCode のインストール

次のURLからインストーラを取得する。

  • Visual Studio Code

既にインストールされているものはそのまま使ってもよいが、最新版を利用すること。

3.1.2 Dev Containers のインストール

VSCode の拡張機能 Dev Containers をインストールする。 これは VSCode から Docker コンテナを起動して、コンテナ内でコードを編集・実行できるようにする拡張機能である。

3.1.3 Python 拡張機能のインストール

合わせて、VSCode の拡張機能 Python をインストールしておく。 コード補完、デバッガ、テストランナーなどがすぐに使えるようになる。

3.2 Docker

Docker はコンテナ型の仮想化ソフトウェアであり、OSの違いを吸収して同一の環境を提供することができる。 VSCode の Dev Containers は Docker コンテナを起動するための拡張機能であり、Docker 自体は別途インストールする必要がある。

3.2.1 Docker のインストール

次のURLからインストーラを取得する。

  • Docker

インストールが済んだら Docker を起動し、ターミナルで次のコマンドを実行してインストールを確認する。

docker --version

3.2.2 Dockerイメージの取得

最後の準備として、本実験用に準備した開発コンテナ設定を取得する。 UNIPA から lab_env.zip をダウンロードして、適当な場所に展開する。 展開すると次のような構成になっている。

lab_env/
├── .devcontainer/
│   ├── devcontainer.json
│   └── Dockerfile
├── reversi_yourid.py     # 各自のプログラム
├── test_reversi.py       # テストファイル
└── pyproject.toml

devcontainer.json には Python のバージョン、必要な拡張機能、必要なパッケージ（pytest, pygame など）が指定されている。 Dockerfile は Python ベースのイメージに対し、本実験で使うパッケージを上書きインストールする。 これらのファイルは原則編集しないこと。

3.3 VSCode から Docker コンテナを起動する

VSCode を起動し、lab_env フォルダを開く。 左下の緑色のアイコンをクリックし、「コンテナーで再度開く（Reopen in Container）」 を選択する。 これにより Docker コンテナが起動し、VSCode がコンテナ内で再起動される。

Warning

最初の起動はイメージのビルドが行われるため非常に時間がかかる。 授業時間のロスを避けるため、次回以降の実験では授業前にあらかじめ起動しておくこと。

コンテナ内で VSCode が起動したら、ターミナルを開いて、次のコマンドで Python の実行環境が構築されていることを確認する。

python --version
pip list | grep -E "pytest|pygame"

Python 3.x.x と表示され、pytest と pygame がインストールされていれば成功である。

3.4 まとめ

  • 本実験では VSCode + Dev Containers + Docker により、統一された Python 実行環境を用いる。
  • 配布される lab_env.zip には Dev Container 設定が含まれており、VSCode で開いて「コンテナーで再度開く」を選ぶだけで環境が立ち上がる。
  • 初回のビルドは時間がかかるため、事前に起動しておくこと。
