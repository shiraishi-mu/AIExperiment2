10  ヒューリスティック評価関数

minimax 法・αβ 法では、終局まで探索できれば勝敗を正確に求められる。 しかし、\(8 \times 8\) のオセロを終局まで全探索することは現実的ではない。 そのため、ある深さで探索を打ち切り、その時点の盤面に評価値を与える必要がある。

完全な解ではないが、問題に関する知識を使って良さそうな値を与える方法をヒューリスティック評価と呼ぶ。 本章では、石数だけでない評価関数を段階的に実装する。

10.1 局面の段階

オセロでは、序盤・中盤・終盤で良い手の性質が変わる。 旧 LISP 版テキストでも、おおよそ次のように分割して考えてきた。

  • 序盤（1～20手）：石を取りすぎず、相手の自由度を下げる。中央の32マスでの陣取り。
  • 中盤（21～40手）：辺の攻防、中割り・中抜き、可動性。
  • 終盤（41手～）：隅の攻防、確定石、できるだけ深く読む。深読みした時点で多くの石を取る。

本実験の完成版では、手数 count に応じて評価の重みを変える。

def eval_node(node, turn, count):
    board = node.board
    if count < 25:
        ...   # 序盤
    elif count < 40:
        ...   # 中盤
    elif count < 60:
        ...   # 終盤前
    else:
        return board_eval(board)   # 終盤

以下の練習で board_eval2 ～ board_eval5 を順に実装し、最後にこれらを組み合わせて eval_node を完成させる。

10.2 位置による評価（board_eval2）

オセロでは、角は非常に強く、角の隣は危険になることが多い。 そこで、盤面上の各マスに点数を与え、自分の石がある場所の点数を加算し、相手の石がある場所の点数を減算する。

10.2.1 例題：盤面の位置点を定義する

BOARD_POINT = [
    [ 30, -12,  0.1, -0.9, -0.9,  0.1, -12,  30],
    [-12, -15, -3,   -3,   -3,   -3,   -15, -12],
    [ 0.1, -3,  0,   -1,   -1,    0,   -3,   0.1],
    [-0.9, -3, -1,   -1,   -1,   -1,   -3,  -0.9],
    [-0.9, -3, -1,   -1,   -1,   -1,   -3,  -0.9],
    [ 0.1, -3,  0,   -1,   -1,    0,   -3,   0.1],
    [-12, -15, -3,   -3,   -3,   -3,   -15, -12],
    [ 30, -12,  0.1, -0.9, -0.9,  0.1, -12,  30],
]

この値は一例である。 角（4隅）に高い点数、その隣（C打ち・X打ちの位置）に大きな負の点数を与えている。 最終課題では、各自で値を変更して強さの変化を調べてよい。

10.2.2 練習26：位置点による評価を実装する

盤面 board と重み omomi を受け取り、位置点に基づく評価値を返す関数 board_eval2(board, omomi) を定義せよ。

def board_eval2(board, omomi):
    count = 0.0
    ...
    return count * omomi

評価の考え方:

  • 先手の石（BOARD_SENTE）なら + BOARD_POINT[row][col] を加える。
  • 後手の石（BOARD_GOTE）なら - BOARD_POINT[row][col] を加える（BOARD_GOTE = -1 を掛けることで実現）。
  • 空きマスは評価に影響させない。

つまり、各マスについて BOARD_POINT[row][col] * board[row][col] を合計し、最後に omomi を掛ければよい。

動作確認:

board = initial_board()
v = board_eval2(board, 3)
assert isinstance(v, float)
# 初期盤面では先手後手の位置点は対称なので 0.0 になるはず
assert v == 0.0

10.3 可動性の評価（board_eval3）

現在打てる手が多いほど、次に選べる選択肢が多い。 逆に、相手の合法手を減らせる手は有利になりやすい。

10.3.1 練習27：合法手の位置を評価する

盤面 board、手番 turn、重み omomi を受け取り、turn が打てる場所を評価する関数 board_eval3(board, turn, omomi) を定義せよ。

def board_eval3(board, turn, omomi):
    count = 0.0
    for row in range(BOARD_SIZE):
        for col in range(BOARD_SIZE):
            if board_movable(board, row, col, turn):
                count += turn * (BOARD_POINT[row][col] + omomi)
    return count

turn が先手なら正方向、後手なら負方向に評価される。 omomi は「とにかく合法手があるだけで嬉しい」という基底点であり、これに位置点を加えて重み付けしている。

Tip設計上のポイント

board_eval3 は turn の合法手だけを評価する。 本当は「自分の合法手数 − 相手の合法手数」を考慮するほうが望ましい。 発展課題として、board_eval3 を turn と change_turn(turn) の両方について計算し、差を取るように改造してみるとよい。

10.4 確定石の評価（board_eval4）

角を取った石、および角から連続して同じ色でつながっている石は、相手に反転されにくい。 これを確定石と呼ぶ。 ここでは簡易的に、4つの角から縦横方向に連続する同色の石を数える。

10.4.1 練習28：1方向の確定石を数える

盤面 board の (row0, col0) から (row_inc, col_inc) 方向へ、手番 turn の石が連続している数を数える関数 kakutei_scan(board, row0, col0, row_inc, col_inc, turn) を定義せよ。 ただし、戻り値は連続数そのものではなく、「連続数 × turn の値」 とする。

def kakutei_scan(board, row0, col0, row_inc, col_inc, turn):
    row = row0
    col = col0
    length = 0
    while 0 <= row < BOARD_SIZE and 0 <= col < BOARD_SIZE and board[row][col] == turn:
        length += turn   # 先手なら +1、後手なら -1
        row += row_inc
        col += col_inc
    return length

このようにすると、後で board_eval4 の中で「先手の確定石は正、後手の確定石は負」になるよう自然に積み上げられる。

動作確認:

board = initial_board()
# 初期盤面では角は空なので、(0, 0) からの確定石は 0
assert kakutei_scan(board, 0, 0, 1, 0, BOARD_SENTE) == 0

# (3, 4) は先手の石。そこから上方向 (-1, 0) には先手1個
assert kakutei_scan(board, 3, 4, -1, 0, BOARD_SENTE) == 0   # 上に空
# 自分自身を含めて数えるので、先手1個分の +1
# ※ 細かい仕様（自身を含めるかどうか）は実装と合わせて確認すること
Warning

kakutei_scan は (row0, col0) 自身も turn の石である場合だけ意味のある値を返す。 角に石が無い場合は呼び出さないようにする（次の board_eval4 で確認している）。

10.4.2 練習29：角からの確定石を評価する

4つの角について、角に石がある場合だけ、縦横の連続石を評価する関数 board_eval4(board, omomi) を定義せよ。

def board_eval4(board, omomi):
    count = 0.0
    if board[0][0] != BOARD_EMPTY:
        turn = board[0][0]
        count += kakutei_scan(board, 0, 0, 1, 0, turn)   # 下方向
        count += kakutei_scan(board, 0, 0, 0, 1, turn)   # 右方向
    if board[7][7] != BOARD_EMPTY:
        turn = board[7][7]
        count += kakutei_scan(board, 7, 7, -1, 0, turn)  # 上方向
        count += kakutei_scan(board, 7, 7, 0, -1, turn)  # 左方向
    if board[0][7] != BOARD_EMPTY:
        turn = board[0][7]
        count += kakutei_scan(board, 0, 7, 1, 0, turn)   # 下方向
        count += kakutei_scan(board, 0, 7, 0, -1, turn)  # 左方向
    if board[7][0] != BOARD_EMPTY:
        turn = board[7][0]
        count += kakutei_scan(board, 7, 0, -1, 0, turn)  # 上方向
        count += kakutei_scan(board, 7, 0, 0, 1, turn)   # 右方向
    return count * omomi
Note

角の座標は \((0, 0)\), \((0, 7)\), \((7, 0)\), \((7, 7)\) の4つ。 各角から内側に向かう2方向（縦・横）について連続石を数えることで、辺に沿った確定石を素朴に評価している。 本来は斜め方向の伝播なども考慮すべきだが、ここでは簡易版にとどめる。

10.5 X打ち・C打ちの評価（board_eval5）

角の斜め隣のマス（X打ち）や、角の辺隣のマス（C打ち）は、角が空いているときに打つと相手に角を取られる原因になりやすい。

10.5.1 練習30：X打ちを減点する

盤面 board、手番 turn、重み omomi を受け取り、X打ちを減点する関数 board_eval5(board, turn, omomi) を定義せよ。

def board_eval5(board, turn, omomi):
    count = 0.0
    if board[1][1] == turn and board[0][0] == BOARD_EMPTY:
        count += turn * -1
    if board[1][6] == turn and board[0][7] == BOARD_EMPTY:
        count += turn * -1
    if board[6][1] == turn and board[7][0] == BOARD_EMPTY:
        count += turn * -1
    if board[6][6] == turn and board[7][7] == BOARD_EMPTY:
        count += turn * -1
    return count * omomi

対象となる X の座標と角の組は次の通り。

Xの位置 対応する角
\((1, 1)\) \((0, 0)\)
\((1, 6)\) \((0, 7)\)
\((6, 1)\) \((7, 0)\)
\((6, 6)\) \((7, 7)\)

X の位置に turn の石があり、対応する角が空きなら、turn * -1 を加算する。 これにより、turn = BOARD_SENTE のときは負、turn = BOARD_GOTE のときは正の値が積まれる（あとで omomi（正値）を掛けるので、最終的に「先手は減点・後手は加点」となる）。

Tip発展課題

C打ち（角の辺隣 \((0, 1), (1, 0), (0, 6), (1, 7), \ldots\)）も同様に減点するよう拡張できる。 最終レポートで、「C打ちを減点項目に追加するとどう変わったか」を実験すると説得力のある考察になる。

10.6 評価関数を組み合わせる

10.6.1 練習31：eval_node() を改良する

これまで作成した評価関数を組み合わせて、eval_node(node, turn, count) を改良せよ。 完成版では次の方針を用いる。

def eval_node(node, turn, count):
    board = node.board
    if count < 25:                  # 序盤
        return (board_eval2(board, 3)
                + board_eval3(board, turn, 20)
                + board_eval4(board, 100)
                + board_eval5(board, turn, 500))
    elif count < 40:                # 中盤
        return (board_eval2(board, 3)
                + board_eval3(board, turn, 40)
                + board_eval4(board, 100))
    elif count < 60:                # 終盤前
        return (board_eval2(board, 3)
                + board_eval3(board, turn, 20)
                + board_eval4(board, 100))
    else:                           # 終盤
        return board_eval(board)

これは一つの基準実装である。 より強いプログラムにするには、重み・局面の分割・評価項目を変更し、実験で比較する必要がある。

10.7 次章以降への接続

本章で扱った評価関数の重み（omomi）は、すべて人間の直感で決めた値である。 これを次のように自動化することができる。

  • 第10章 手動チューニング：実験を繰り返して人間が改良する（次章）。
  • 第11章 メタヒューリスティクス：山登り法・焼きなまし法・遺伝的アルゴリズムなどの探索アルゴリズムで重みを自動最適化する。
  • 第12章 機械学習：自己対戦データから評価関数そのものを学習する。

本実験では、まず本章の手動の評価関数で動くプログラムを完成させ、その後の章でこれらの自動化を体験する。 それぞれの章は AI の歴史的発展（推論 → 最適化 → 学習）に対応しており、同じ問題を異なる視点で解くことの面白さを味わえるはずである。

10.8 まとめ

  • 評価関数はオセロの強さを決める中核である。
  • 位置点・可動性・確定石・X打ちなど、ゲームの知識を分けて関数化し、重み付き和で組み合わせるのが基本戦略。
  • 序盤・中盤・終盤で重みを変えることで、局面に応じた指針に切り替えられる。
  • 改良の効果は実験で確かめて、結果として記録する習慣をつける。
  • 手動チューニングで強い AI を作るのは限界がある。次章以降では、自動最適化や機械学習でこの限界を超える方法を学ぶ。
