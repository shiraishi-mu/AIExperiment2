9  αβ 枝刈り

minimax 法は、すべての子節点を探索してから最善手を決める。 しかし、探索の途中で「この枝は最終的に選ばれない」と分かる場合がある。 そのような枝の探索を省略する方法がαβ 枝刈り（alpha-beta pruning） である。

αβ 法は、同じ評価関数・同じ子節点の順序であれば minimax 法と同じ手を返す。 しかし、探索する節点数を減らせるため、同じ計算予算でより深い読みが可能になる。

9.1 αとβ

αβ 法では、探索中に次の2つの値を持ち回る。

  • alpha：先手がこれまでに保証できている下限値
  • beta：後手がこれまでに保証できている上限値

探索中に alpha >= beta になったら、それ以上その枝を調べても親の選択は変わらない。 そのため、そこで探索を打ち切る。

9.2 αβ 法の本体

9.2.1 練習20：alpha_beta() を定義する

節点 node、手番 turn、手数 count、読みの深さ depth、下限 alpha、上限 beta を受け取り、最善手に対応する節点を返す関数 alpha_beta(node, turn, count, depth, alpha, beta) を定義せよ。

def alpha_beta(node, turn, count, depth, alpha, beta):
    state = node_state(node, turn, count)
    ...

基本構造は minimax() と同じである。

  • 終局なら、node.value = state を設定して node を返す。
  • 深さ限界なら、node.value = eval_node(node, turn, count) を設定して node を返す。
  • パスなら、手番を交代して同じ深さで再帰呼び出しする。
  • それ以外なら、子節点を生成して alpha_beta_children() で選ぶ。

alpha と beta は、最初はまだ値がないため None を渡す（呼び出し側で alpha_beta(node, turn, count, depth, None, None) のようにする）。

9.3 子節点の探索

9.3.1 練習21：alpha_beta_children() を定義する

children を順に探索し、αβ 枝刈りを行いながら最善の節点を返す関数 alpha_beta_children(children, turn, count, depth, alpha, beta) を定義せよ。

def alpha_beta_children(children, turn, count, depth, alpha, beta):
    new_turn = change_turn(turn)
    new_depth = depth - 1
    new_count = count + 1
    val = None
    result_node = None
    for child in children:
        v = alpha_beta(child, new_turn, new_count, new_depth, alpha, beta).value
        if turn == BOARD_SENTE:
            if val is None or v > val:
                val = v
                result_node = Node(child.board, child.turn, child.row, child.col, v)
            if alpha is None or v > alpha:
                alpha = v
        elif turn == BOARD_GOTE:
            if val is None or v < val:
                val = v
                result_node = Node(child.board, child.turn, child.row, child.col, v)
            if beta is None or v < beta:
                beta = v
        if alpha is not None and beta is not None and alpha >= beta:
            break
    return result_node

各部分の意味を確認する。

  • 先手番では、子節点の最大評価値を選び、alpha（下限）を更新する。
  • 後手番では、子節点の最小評価値を選び、beta（上限）を更新する。
  • 最後に alpha >= beta になっていたらループを抜けて以降の子節点を探索しない（枝刈り）。
Tip注意：None の扱い

最初は alpha も beta も None である。 比較の前に必ず is not None を確認する。 これは「まだ何も値がない」状態を表すためのものである。 最初の子の評価値が決まれば、alpha（または beta）はその値で初期化される。

9.4 コンピュータの着手に組み込む

9.4.1 練習22：play_by_machine() をαβ法に切り替える

play_by_machine() で呼び出す探索関数を minimax() から alpha_beta() に変更する。

def play_by_machine(board, turn, count, depth):
    node = Node(board, turn, 0, 0, 0.0)
    # best_node = minimax(node, turn, count, depth)
    best_node = alpha_beta(node, turn, count, depth, None, None)
    row, col = best_node.row, best_node.col
    board_move(board, row, col, turn)
    print(f"({count}) My move is {row + 1} {col + 1}.")

minimax 法と αβ 法は、同じ評価関数と同じ子節点の順序であれば、同じ手を選ぶ。 結果が大きく異なる場合は、value の更新、手番の交代、パス処理を確認すること。

Note切り替えを実験で残すコツ

両方の関数を残して切り替え可能にしておくと、後で比較しやすい。 たとえばモジュール変数 USE_ALPHA_BETA = True を導入して、

def play_by_machine(board, turn, count, depth):
    node = Node(board, turn, 0, 0, 0.0)
    if USE_ALPHA_BETA:
        best_node = alpha_beta(node, turn, count, depth, None, None)
    else:
        best_node = minimax(node, turn, count, depth)
    ...

のように書くと、評価関数を変えずに探索手法だけ切り替えて比較できる。

9.5 探索効率の比較

9.5.1 練習23：呼び出し回数を数える

minimax 法と αβ 法の効率の違いを定量的に確認するため、探索関数が呼ばれた回数を数える。 Python では、モジュール内のグローバル変数を使うと簡潔に書ける。

NODE_CHECK_COUNT = 0

def node_check_begin():
    global NODE_CHECK_COUNT
    NODE_CHECK_COUNT = 0

def node_check():
    global NODE_CHECK_COUNT
    NODE_CHECK_COUNT += 1

def node_check_end():
    print(f"{NODE_CHECK_COUNT} nodes checked.")

minimax() と alpha_beta() の先頭に node_check() を呼び、play_by_machine() の前後で node_check_begin()、node_check_end() を呼ぶ。

def play_by_machine(board, turn, count, depth):
    node_check_begin()
    node = Node(board, turn, 0, 0, 0.0)
    best_node = alpha_beta(node, turn, count, depth, None, None)
    node_check_end()
    ...

9.5.2 練習24：実験の記録

比較する条件を揃えて、minimax 法と αβ 法の探索節点数・実行時間を表に記録せよ。

探索深さ 手法 探索節点数 実行時間 [s] 選んだ手
3 minimax
3 αβ
4 minimax
4 αβ
5 minimax
5 αβ

実行時間は time.perf_counter() を使う。

import time

start = time.perf_counter()
best_node = alpha_beta(node, turn, count, depth, None, None)
elapsed = time.perf_counter() - start
print(f"{elapsed:.3f} sec")

9.5.3 練習25：考察

次の点について考察せよ。

  • 浅い深さでは αβ の効果が小さい理由。
  • 深い深さでは αβ の効果が大きくなる理由。
  • 子節点の探索順序を変えると効果はどう変わるか。

確認できたら、Git でコミットしておくこと。

git add reversi_yourid.py test_reversi.py
git commit -m "implement alpha-beta pruning (ex.20-25)"

9.6 まとめ

  • αβ 法は、minimax 法と同じ最適解を返しつつ、探索する節点数を減らす。
  • 効果が出るのは「ある程度深く読む」場合。理想的な枝刈りでも計算量は約 \(1/2\) にしかならないが、その分深く読むことで強さは大きく向上する。
  • 子節点の順序が良ければ枝刈りの効果はさらに大きくなる。これを生かす工夫は応用課題として考えられる。
