8  minimax 法によるゲーム木探索

本章では、人間が入力する代わりに、コンピュータが次の手を選ぶ仕組みを実装する。 現在の盤面から合法手をすべて展開すると、1手先の盤面の集合が得られる。 さらに、その各盤面から相手の合法手を展開していくと、ゲーム木が得られる。

本章の到達目標は、minimax 法を用いてコンピュータが手を選べるようにすることである。

8.1 節点のデータ構造

ゲーム木の節点には、盤面、その盤面に至るために打った手、評価値などを保存する。 Python ではクラスを使って表現する。

8.1.1 例題：Node クラスを定義する

class Node:
    def __init__(self, board, turn, row, col, value):
        self.board = board
        self.turn = turn
        self.row = row
        self.col = col
        self.value = value

各属性の意味は次の通りである。

  • board: その節点に対応する盤面（リストのリスト）。
  • turn: その盤面に至る手を打った手番（BOARD_SENTE または BOARD_GOTE）。
  • row, col: その盤面に至るために置いた位置（0始まり）。
  • value: その節点に与えられた評価値。
Note

ここでの turn は「この節点の盤面で次に打つ手番」ではなく、「この盤面に至るために直前に打った手番」である点に注意する。 初期局面ではまだ着手がないので、turn には便宜上 BOARD_SENTE、row/col には 0 などの仮の値を入れておく。

root = Node(initial_board(), BOARD_SENTE, 0, 0, 0.0)

8.2 節点の操作関数

探索に必要な操作は大きく3つである。

  • 節点が終局、パス、継続中のどれであるかを判定する。
  • 節点を評価する。
  • 節点から子節点を生成する。

8.2.1 例題：節点の状態を返す

節点 node の盤面に対して、手番 turn の状態を返す関数 node_state(node, turn, count) を定義する。 count は後で局面の段階に応じて評価関数を切り替えるために受け取る引数で、現時点では使わない。

def node_state(node, turn, count):
    return board_state(node.board, turn)

8.2.2 例題：節点を評価する

まずは単純化のため、探索を打ち切った節点の評価値として石数評価 board_eval() を使う。

def eval_node(node, turn, count):
    return board_eval(node.board)

第9章で、この関数をより強い評価関数に差し替える。

8.2.3 練習15：子節点を生成する

手番 turn において、節点 parent から打てるすべての手を調べ、その手を打った後の子節点のリストを返す関数 expand_node(parent, turn) を定義せよ。

def expand_node(parent, turn):
    children = []
    ...
    return children

実装上の注意:

  • parent.board を直接変更してはいけない。
  • 合法手ごとに new_board = [row[:] for row in parent.board] で盤面をコピーする。
  • コピーした盤面に board_move(new_board, row, col, turn) を適用する。
  • Node(new_board, turn, row, col, 0.0) を作って children に追加する。

動作確認:

root = Node(initial_board(), BOARD_SENTE, 0, 0, 0.0)
children = expand_node(root, BOARD_SENTE)
assert len(children) == 4
assert all(isinstance(child, Node) for child in children)

8.3 minimax 法の実装

minimax 法では、先手は評価値を最大化し、後手は評価値を最小化する。 評価値は「先手が有利な盤面ほど大きく、後手が有利な盤面ほど小さい」ように設計しているからである。

探索の基本方針は次の通りである。

  • 終局なら、その盤面の評価値を返す。
  • 読みの深さが \(0\) なら、評価関数で評価した値を返す。
  • パスなら、手番だけを交代して同じ深さで探索を続ける。
  • 先手番なら、子節点の中で評価値が最大のものを選ぶ。
  • 後手番なら、子節点の中で評価値が最小のものを選ぶ。

8.3.1 練習16：minimax 本体を定義する

節点 node、手番 turn、手数 count、読みの深さ depth を受け取り、最善手に対応する節点を返す関数 minimax(node, turn, count, depth) を定義せよ。

def minimax(node, turn, count, depth):
    state = node_state(node, turn, count)
    ...

以下の場合分けを行う。

  • state が float なら終局である。node.value = state を入れて node を返す。
  • depth == 0 なら、node.value = eval_node(node, turn, count) を入れて node を返す。
  • state == "pass" なら、手番を交代して同じ節点から探索を続ける（depth は変更しない）。
  • それ以外なら、expand_node() で子節点を作り、minimax_children() に渡す。

float かどうかは次のように判定する。

if isinstance(state, float):
    ...

8.3.2 練習17：子節点の中から最善手を選ぶ

children の各節点について再帰的に minimax() を呼び出し、手番に応じて最大または最小の値を持つ節点を返す関数 minimax_children(children, turn, count, depth) を定義せよ。

def minimax_children(children, turn, count, depth):
    new_turn = change_turn(turn)
    new_depth = depth - 1
    new_count = count + 1
    val = None
    result_node = None
    for child in children:
        v = minimax(child, new_turn, new_count, new_depth).value
        if val is None or (turn == BOARD_SENTE and v > val) or (turn == BOARD_GOTE and v < val):
            val = v
            result_node = Node(child.board, child.turn, child.row, child.col, v)
    return result_node

戻り値は、評価値そのものではなく、次に打つ手に対応する Node とする。 返す節点の value には、探索で得られた評価値を保存する。

動作確認:

root = Node(initial_board(), BOARD_SENTE, 0, 0, 0.0)
best = minimax(root, BOARD_SENTE, 1, 2)
assert isinstance(best, Node)
assert board_movable(initial_board(), best.row, best.col, BOARD_SENTE)

8.4 コンピュータの着手

8.4.1 練習18：コンピュータが石を置く

盤面 board、手番 turn、手数 count、読みの深さ depth を受け取り、minimax 法で選んだ手を盤面に反映する関数 play_by_machine(board, turn, count, depth) を定義せよ。

def play_by_machine(board, turn, count, depth):
    root = Node(board, turn, 0, 0, 0.0)
    best_node = minimax(root, turn, count, depth)
    row, col = best_node.row, best_node.col
    board_move(board, row, col, turn)
    print(f"({count}) My move is {row + 1} {col + 1}.")

best_node.row, best_node.col が次に打つ位置である。 1始まりに直して表示する点に注意する。

8.5 人間対コンピュータ、コンピュータ対コンピュータ

8.5.1 練習19：8×8盤で動作を確認する

第6章の練習13で定義した othello(sente_gote, yomi_depth) は、play_by_machine を呼び出すように作っておけば、本章の play_by_machine がそのまま使われる。 8×8盤で次のすべてが動くことを確認せよ。

othello(BOARD_GOTE, 3)    # 人間先手 vs コンピュータ後手
othello(BOARD_SENTE, 3)   # コンピュータ先手 vs 人間後手
othello(BOARD_EMPTY, 2)   # コンピュータ vs コンピュータ

確認すべき項目は次の通り。

  • 初期盤面で先手の合法手が4つである。
  • 人間対人間で最後まで実行できる（前章で確認済み）。
  • 人間対コンピュータで最後まで実行できる。
  • コンピュータ対コンピュータで最後まで実行できる。
  • パスが発生しても手数の扱いが破綻しない。
  • 探索深さを大きくすると思考時間が伸びる（深さ3と5を比較）。

確認できたら、Gitでコミットしておくこと。

git add reversi_yourid.py test_reversi.py
git commit -m "implement minimax reversi player (ex.15-19)"

8.6 まとめ

  • ゲーム木の節点を Node クラスで表現し、expand_node() で子節点を作り、minimax() で再帰的に最善手を探索した。
  • パスは「手番だけ交代して同じ深さで再帰呼び出し」で扱う。
  • 終局は state が float であることで判定する。
  • 探索深さを大きくすると、計算時間は指数的に増える。次章ではこれを抑える αβ 枝刈りを導入する。
