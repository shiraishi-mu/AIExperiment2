6  ゲームと探索

6.1 ゲームとは

ゲーム（Game） という言葉は、複数のプレイヤーがルールに従って行う競技や遊びを指すことが多いが、一人で行うゲームも含まれる。 将棋、チェス、麻雀などのボードゲーム・カードゲームから、スポーツまでが「ゲーム」に含まれる。 オリンピックが Olympic Games と呼ばれることからも分かるように、ゲームの範疇は非常に広い。 オリンピック競技には個人記録を求める種目もあるが、どこまで頑張るか、頑張らないかは他競技者との駆け引きであり、広く「ゲーム」とみなせる。

このようなゲームの抽象的な要素をモデル化したのがゲーム理論（Game Theory） である。 ゲーム理論は、フォン・ノイマンとモルゲンシュテルンによって1944年に発表された『ゲームの理論と経済行動（Theory of Games and Economic Behavior）』で初めて体系化された。 遊びとしてのゲームというよりは、経済学・社会学などの分野での人間の行動を分析するための理論である。 今日ではネットワーク科学とも結びつき、社会現象や経済予測などの分析にも用いられている。

以下では、このゲーム理論に応じた定義に基づき、概要を説明する。

6.2 ゲームの分類

ゲームを整理し解法を導くには、対象を分類する必要がある。 分類の代表的なものさしを以下に示す。

  • プレイヤーの人数：将棋などは二人、麻雀は四人。人数が増えるほど複雑になる。
  • 情報の完全性：将棋などは盤面や持ち駒などの状態が全て把握できる。一方ポーカーでは相手の手札が見えない。情報が欠落するほどアルゴリズムは非決定的になる。
  • 状態の有限性：状態の多様性が有限かどうか。状態数が多いほど処理量も大きい。
  • コスト総和の一定性：プレイヤーの利害が互いに相反するかどうか。協力で利得が増やせるものを非零和ゲーム、誰かの損が必ず別の誰かの得になるものを零和ゲームと呼ぶ。

本実験で扱うオセロゲームは、上記の分類によると 二人完全情報有限零和ゲーム である。

6.3 ゲームの解析

ゲームの本質を解明するには、次の二点が分かればよい。

  • プレイヤーはどう行動すべきか？（ゲームのアルゴリズム）
  • ゲームの最終結果はどうあるべきか？（最終目標）

すなわち、各局面で最もふさわしい一手を選び、それを繰り返して最終的に勝利へと導くことが望まれる。

ここで問題となるのが、最終目標である局面の状態と、中間の局面の状態とのギャップである。 ある程度の大きさのゲームでは、状態数は計算機のメモリに乗らないほど膨大になる。 たとえばオセロの状態数は \(10^{28}\) という報告があり、終局までの全状態を計算したり保存したりするのは現実的ではない。 結局、ある局面で次の手を決めるとき、最終的にどのような終局になるかが分からないまま判断しなければならない。

このような状況で人間（および計算機）が取る戦略は、有限の数手だけ先を読み、その範囲内で最良の手を選ぶ ことである。 最良の手を選ぶには、そのゲームに応じた評価関数が必要であり、ヒューリスティックなアルゴリズム（ゲームの世界では定跡と呼ばれる）を導入することも有効である。

以下では、二人完全情報有限零和ゲームに対する一般的な手法を説明する。

6.4 状態の表現と探索木

オセロは二人が交互に手を打ち合うゲームである。 先手の最初の打ち手を第1手とすると、後手が第2手、また先手が第3手……というふうに、手数の偶奇でどちらが打つかが決まる。

ある手を打った直後の状態を頂点とし、次に打てる手を枝として展開していくと、ゲームの状態遷移を表す木が出来上がる（図 Figure 6.1）。

ゲームを計算機で解く場合、このようなゲーム木を作り、その上で最適な解を探索する。 厳密にはゲーム状態は木ではなくグラフになるが、これは、循環劫や千日手をルールで排除したり、同じ状態を別の頂点で扱うことで木として表現できる。

ゲームの進行は、ゲーム木の根（root）から葉（leaf）へ向かう状態遷移として表せる。 オセロで自分が勝利するには、葉の状態で自分の石数を相手より多くすることが目標になる。 すなわち、最終局面に到達する可能性の高い手を採用することが要求される。

なお、現在の手から葉まで辿って自分が勝つことが確定できることを「必勝法が存在する」という。 たとえばコネクトフォー（先手必勝）、五目並べ（先手必勝）、6×6 オセロ（後手必勝）などが知られている。

次に、ゲーム木を使って次の一手を決定する一般的手法を説明する。

6.5 minimax 法

この種のゲームは、自分と相手の成績争いである。 自分が不利な手を打つと、自動的に相手が有利になる。 よって、相手はできるだけ相手に有利な手を打つし、自分はできるだけ自分に有利な手を打つはずである。

これらをまとめると、

  • 自分の番では最大利得の手を選ぶ（max）
  • 相手の番では自分にとって最小利得の手を選ぶ（min）

を交互に繰り返すことになる（図 Figure 6.2）。

これがミニマックス法（minimax） であり、先手・後手の双方が互いに最善を尽くすと仮定したうえで、自分にとって最も有利になるように手を選ぶ戦略である。

具体的には次のように評価する。 ゲーム木の葉では勝敗が決まっているので、

  • 先手が勝つ葉に \(1\)
  • 後手が勝つ葉に \(-1\)
  • 引き分けの葉に \(0\)

を与える。 それ以外の頂点（葉に至る前）は、

  • 先手番なら子の値の最大を採用
  • 後手番なら子の値の最小を採用

と再帰的に計算する。 最終的に根（root）に \(1\) がつけば先手必勝、\(-1\) がつけば後手必勝、\(0\) なら引き分けとなる。

このミニマックス法を計算機で実現するには、木全体を後順走査（post-order） しながら各頂点の評価値を決めればよい。 後順走査の擬似コードを示す。

def post_order(n, level):
    """後順走査の枠組み（擬似コード）"""
    if n に子供が存在する:
        n の子供を n_1, n_2, ..., n_k とする
        for i in range(1, k + 1):
            post_order(n_i, level + 1)
    else:
        n に対する処理を行う

これを minimax に拡張すると次のようになる。

def minimax(node, turn, depth):
    """minimax 法（擬似コード）"""
    if node の状態 == 先手が勝ち:
        return 1
    elif node の状態 == 後手が勝ち:
        return -1
    elif node の状態 == 引き分け:
        return 0

    if depth == 0:
        return 局面の評価関数(node)

    現在の局面から1手進めた状態を a_1, ..., a_k とする
    if turn == 先手:
        return max(minimax(a_i, 次の手番, depth - 1) for a_i in 子)
    else:
        return min(minimax(a_i, 次の手番, depth - 1) for a_i in 子)

このアルゴリズムは有用ではあるが、探索しなくてもよい頂点まで探索してしまうという無駄がある。 限られた計算機の記憶領域や制限時間のもとでは、この無駄を抑える工夫が望まれる。

6.6 αβ 枝刈り（先取り）

ミニマックス法で行われる無駄な探索のうち、「この枝はどうせ親の選択に影響しない」と判明した時点で打ち切るのがαβ法（alpha-beta pruning） である。

αβ法は、

  • 先手の番でこれまでに保証できる下限値 \(\alpha\)
  • 後手の番でこれまでに保証できる上限値 \(\beta\)

の二つを管理し、探索中に \(\alpha \geq \beta\) になった枝の探索をスキップする（図 Figure 6.3）。

Note

αβ法は、深く読む場合に効果が大きい。 1手読みでは枝刈りは全く起きず、むしろ余分な比較が増える。 理想的な枝刈りでも計算量はおよそ半分にしかならないが、それを使って読みの深さを2倍に増やせれば、読みの確かさは飛躍的に向上する。

αβ法の擬似コードを示す。詳細は第8章で扱う。

def alpha_beta(node, turn, depth, alpha, beta):
    """αβ法（擬似コード）"""
    if node の状態 == 先手が勝ち:
        return 1
    elif node の状態 == 後手が勝ち:
        return -1
    elif node の状態 == 引き分け:
        return 0
    if depth == 0:
        return 局面の評価関数(node)

    現在の局面から1手進めた状態を a_1, ..., a_k とする
    for i in range(1, k + 1):
        if alpha is not None and beta is not None and alpha >= beta:
            break
        v = alpha_beta(a_i, 次の手番, depth - 1, alpha, beta)
        if turn == 先手 and (alpha is None or v > alpha):
            alpha = v
        if turn == 後手 and (beta is None or v < beta):
            beta = v
    return alpha if turn == 先手 else beta

なお、ミニマックス法もαβ法も、「相手が間違える可能性まで考えて大胆に攻める」というような戦略はとらない。 相手が常に最善を尽くすという仮定のもとで動くため、ある意味消極的な戦略であるとも言える。

6.7 評価値（コスト）の計算

ミニマックス法を使ってすべての終局から根まで遡れば、原理的にはゲームに勝てる。 しかし、計算機の容量・時間の制約から、ゲーム全体を解析できるのは小さなゲームに限られる。 たとえば将棋の場合、ゲームが終了するまでに100手以上かかり、各局面で10種類以上の手が考えられるので、頂点数は少なくとも \(10^{100}\) になる。 これを全て計算するのは到底不可能である。 オセロでも状態数は \(10^{28}\) ほどあるので、実際にはある程度の深さまで読んで打ち切る。

すなわち、完全な勝敗が決定しない段階で何らかの評価値を付ける必要がある。 評価値は、先手有利から後手有利まで \(-1 < \text{評価値} < 1\) の実数値とすることが多い。 この値の付け方にはゲームに関する知識が不可欠であり、たとえば将棋なら駒の損得、王将の守備、攻め駒の体勢などを評価する。 オセロでは、角の有無、辺の安定性、可動性などを使う。 これらを定量化したものが評価関数（evaluation function） であり、本実験の主題のひとつである。

また、読みを打ち切る深さも重要な要素である。 深く読むほど評価が正確になるのは自明だが、計算時間も指数的に増加する。 評価関数の精度と探索深さは組み合わせて設計する必要がある。

6.8 まとめ

  • 二人完全情報有限零和ゲームは、ゲーム木と minimax 法によって扱える。
  • 状態空間が大きすぎる場合は、有限の深さで打ち切り、評価関数で局面を数値化する。
  • 探索の無駄は αβ 枝刈りで削れる。
  • 評価関数の設計と探索の深さの選択が、強さを決める二大要素である。
