14  対戦実験と GUI

旧 LISP 版テキストでは、C 言語のクライアントと LISP の思考プログラムをファイル経由で接続し、ロックファイルで排他制御を行いながら対戦していた。 これは、当時 LISP では GUI を扱いにくかった事情と、複数言語のシステムを協調させる演習の意味があった。

2026 年度の本実験では、Python でオセロ本体と思考部分を同じプログラムとして実装するため、旧版の C/LISP 間通信、LISP のコンパイル、AI.c の作成は扱わない。 代わりに、Python で実装した reversi_yourid.py を使った対戦実験のやり方と、付属の GUI 対戦ツール reversi-gui.py の使い方を紹介する。

14.1 CUI での実行

reversi_yourid.py の末尾には、直接実行されたときだけゲームを開始する処理を書く。

if __name__ == "__main__":
    othello(BOARD_EMPTY, 3)

この例では、先手も後手もコンピュータが担当し、探索深さ3で実行される。 人間が先手、コンピュータが後手を担当する場合は、次のように変更する。

if __name__ == "__main__":
    othello(BOARD_GOTE, 5)

ターミナルで実行する。

python reversi_yourid.py

14.2 対戦ログの保存

対戦結果をレポートに使うため、実行結果をファイルに保存する。

python reversi_yourid.py > log_depth3.txt

人間が着手する場合は対話的に動くため、画面を見ながら必要な部分を記録してよい。 コンピュータ同士の対戦では、ログを保存しておくと後で比較しやすい。

14.3 条件を変えた比較

評価関数や探索深さを変更するときは、条件を明確にして比較する。

othello(BOARD_EMPTY, 2)
othello(BOARD_EMPTY, 3)
othello(BOARD_EMPTY, 4)
Note

探索深さを大きくすると必ず強くなるとは限らない。 評価関数が不適切な場合、深く読んでも誤った局面を高く評価してしまうことがある。 評価関数と探索深さは組み合わせて評価する必要がある。

14.4 GUI による対戦：reversi-gui.py

codes/reversi-gui.py には、pygame を使ったオセロの GUI が用意されている。 人間対 AI、人間対人間、AI 対 AI の3通りで遊べる。

14.4.1 起動方法

reversi-gui.py は、同じディレクトリにある reversi.py を import して使う。 学生が改良した reversi_yourid.py で動かすには、いくつか方法がある。

  1. シンボリックリンクを貼って reversi.py として読ませる。

    ln -s reversi_yourid.py reversi.py
python reversi-gui.py

  2. reversi-gui.py の冒頭の import reversi as rv を import reversi_yourid as rv に書き換える。

TipDev Container での pygame 表示

Docker コンテナ内で pygame のウィンドウを表示するには、X サーバや WSLg、もしくは VSCode の Remote Display 拡張のような仕組みが必要になる。 表示できない環境では、CUI 版（reversi_yourid.py）で動作確認してから、ホストOS上で GUI を起動する形でも構わない。

14.4.2 操作

  1. 起動すると、先手と後手をそれぞれ「HUMAN」「AI」のどちらにするかを選ぶ画面が表示される。
  2. ボタンを押して切り替え、START を押すとゲーム開始。
  3. 人間の番では、ボードのマスをクリックして石を置く。
  4. AI の番では、自動で alpha_beta を呼んで手を決める。
  5. ゲームが終わると、勝敗とスコアが下部に表示される。

14.4.3 内部構造

reversi-gui.py は、本書で実装してきた次の関数をそのまま使っている。

  • rv.initial_board()、rv.board_movable()、rv.board_move()：ルール部分。
  • rv.board_state()、rv.board_number_check()：終局判定とスコア。
  • rv.Node、rv.alpha_beta()：思考部分。

つまり、本書で作る関数のインタフェース（引数と戻り値）を保っている限り、改良した評価関数や探索手法でそのまま GUI 対戦できる。 最終課題で自分の AI を強化したら、ぜひ GUI 上で人間と対戦してみてほしい。

14.5 Git で実験を管理する

評価関数を試すときは、変更ごとにコミットしておく。

git status
git add reversi_yourid.py
git commit -m "adjust corner weights"

うまくいかなかった変更も、結果を記録しておくとレポートで説明しやすい。 不要になった実験コードを残す場合は、関数名やコメントで目的が分かるようにする。

ブランチを使うと、実験ごとに枝分かれを作って後で比較できる。

git switch -c experiment-stronger-corner    # 新しいブランチで実験
# ... コードを変更 ...
git commit -m "increase corner weight to 200"
# 良ければ main にマージ、悪ければブランチを破棄

14.6 提出前の確認

提出前に次を確認する。

  • python reversi_yourid.py でエラーなく実行できる。
  • 先手・後手・コンピュータ同士のいずれかで最後まで対戦できる。
  • python -m pytest がエラーなく通る。
  • 最終的に使っている評価関数がレポートで説明されている。
  • Git のコミット履歴から、段階的に開発したことが分かる。

14.7 発展：外部対戦サーバとの接続

外部の対戦サーバや自動採点器に接続する場合は、授業で指定された入出力仕様に合わせて、reversi_yourid.py の上に薄いインタフェース層を追加する。 本書で作成した合法手判定・盤面更新・探索・評価関数はそのまま再利用できる。

旧 LISP 版で扱われていたファイル経由のプロトコル（/tmp/ban.iXXXXX.saki などのデータファイルとロックファイル）も、Python であれば pathlib と time.sleep を使って素直に書ける。 ただし、本年度の必修範囲ではここまで踏み込まないので、興味のある学生向けの発展課題として位置付ける。

14.8 まとめ

  • 対戦実験は条件をそろえて行い、ログを保存する。
  • reversi-gui.py は本書で作成した関数をそのまま使う pygame ベースの GUI である。インタフェースを保っていれば自分の改良 AI でそのまま動く。
  • 実験ごとに git commit し、開発履歴を残すことが最終評価で重要になる。
