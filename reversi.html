7  オセロゲームの基本実装

本章では、Pythonでオセロゲームの基本機能を実装する。 ここで作成する関数は、後の章でゲーム木探索、minimax法、αβ法、評価関数の改良にそのまま再利用する。

各課題は、reversi_yourid.py に順に追加していく。 動作確認用の assert 文がついている練習は、そのまま test_reversi.py にテストとして追加し、pytest で通ることを確認しながら進めること。

7.1 本章の到達目標

  • 盤面を表現する。
  • 手番を交代する。
  • 盤面を表示する。
  • 合法手を判定する。
  • 石を反転する。
  • パス、終局、勝敗を判定する。
  • 人間対人間のオセロゲームを動かす。

7.2 オセロゲームのルール

オセロは、\(8 \times 8\) の盤上で先手と後手が交互に石を打ち、最終的に石の数の多い方が勝ちとなる二人ゲームである。 本実験では、先手を 1、後手を -1、空きを 0 として盤面を表現する。

ルールを確認しておく。

  1. 最初に盤の中央に先手・後手の石を2個ずつ交差するように置く。
  2. 相手の石をはさめる場所にだけ自分の石を置ける。
  3. はさんだ相手の石はすべて自分の石に反転する。
  4. 先手・後手が交互に1石ずつ打つ。
  5. 打てる場所がない場合はパスとなる。
  6. 打てる場所がある場合、自分の意思でパスすることはできない。
  7. 盤面がすべて埋まるか、両者とも打てなくなれば終了する。

「はさむ」とは、自分の打った石と既に盤面にある自分の石が、縦・横・斜めの8方向のいずれかに並び、その間に相手の石が1つ以上連続して存在することをいう。

Tip

旧 LISP 版の教材では、最初に \(4 \times 4\) の盤で動作確認をしてから \(8 \times 8\) に拡張する流れを採用していた。 本書では BOARD_SIZE を定数で変更すれば \(4 \times 4\) にも切り替えられるようコードを書く。 小さな盤でデバッグすると合法手や反転を目視で確認しやすい。

7.3 盤面の表現

盤面はリストのリストで表す。 外側のリストが行、内側のリストが列である。 Pythonの添字は 0 から始まるが、画面表示と入力では人間に分かりやすいように 1 から 8 の番号を使う。

board[row][col]

例えば、board[3][4] は4行5列目を表す。

7.3.1 例題：定数を定義する

盤面の大きさ、石の値、表示文字、8方向を表す定数を定義する。

BOARD_SIZE = 8
BOARD_SENTE = 1
BOARD_GOTE = -1
BOARD_EMPTY = 0

BOARD_SENTE_STR = "*"
BOARD_GOTE_STR = "o"
BOARD_EMPTY_STR = "-"

BOARD_SYMBOL = {
    BOARD_SENTE: BOARD_SENTE_STR,
    BOARD_GOTE:  BOARD_GOTE_STR,
    BOARD_EMPTY: BOARD_EMPTY_STR,
}

BOARD_DIRECTIONS = [
    (0,  1),   # 右
    (1,  1),   # 右下
    (1,  0),   # 下
    (1, -1),   # 左下
    (0, -1),   # 左
    (-1, -1),  # 左上
    (-1, 0),   # 上
    (-1, 1),   # 右上
]

本資料では、先手を *、後手を o と表示する。 BOARD_SENTE と BOARD_GOTE は、盤面上の石を表す値であると同時に、手番を表す値としても使う。

7.5 盤面の走査

合法手を判定するには、石を置いた場所から8方向を調べる必要がある。 ある方向について、次の3通りを区別する。

  1. 自分の石が出る前に盤面の外に出た。
  2. 自分の石が出る前に空きマスに到達した。
  3. 相手の石を1つ以上はさんだ後、自分の石に到達した。

このうち、相手の石を反転できるのは3の場合だけである。

board の (row0, col0) に turn の石を置くと仮定し、(row_inc, col_inc) 方向に反転できる相手の石の数を返す関数 board_scan(board, row0, col0, row_inc, col_inc, turn) を定義せよ。

def board_scan(board, row0, col0, row_inc, col_inc, turn):
    ...

動作確認:

board = initial_board()
assert board_scan(board, 2, 3, 1, 0, BOARD_SENTE) == 1
assert board_scan(board, 2, 2, 1, 1, BOARD_SENTE) == 0
assert board_scan(board, 0, 0, 1, 1, BOARD_SENTE) == 0
Tipヒント

(row0, col0) そのものは、これから石を置く空きマスである。 したがって、走査は (row0 + row_inc, col0 + col_inc) から始める。 ループの中では「枠外」「空きマス」「自分の石」「相手の石」の4ケースを区別する。 自分の石に到達したらそれまでに数えた相手石の数を返す。 枠外や空きマスに到達したら、はさめなかったので 0 を返す。

石数に基づく評価関数 board_eval(board) を定義せよ。 評価値は次の式で求める。

この値は、先手が有利なら正、後手が有利なら負、互角なら \(0\) になる。 分母が \(0\) になるとき（理論上は起きない）は 0.0 を返すようにしておく。

    ...

動作確認:

board = initial_board()
assert board_eval(board) == 0.0

Warning戻り値の型に注意

board_state() は None、str（“pass”）、float（評価値） の3種類の型を返す。 これを受ける側では、isinstance(state, float) で終局を判定する。 評価値が偶然 0.0 であっても isinstance(state, float) は True になるので、if state: のような偽値チェックは使わない。

満たすべき条件は次の通りである。

    continue
try:
    row = int(parts[0]) - 1
    col = int(parts[1]) - 1

7.9 人間対人間のゲーム

7.9.1 練習13：汎用ゲーム関数を定義する

第7章でコンピュータ対コンピュータの実装を行うことを見据えて、ここでは汎用のゲーム関数を一つ定義する。 othello(sente_gote, yomi_depth) は次の通りに振る舞う。

sente_gote 意味
BOARD_SENTE コンピュータが先手、人間が後手
BOARD_GOTE 人間が先手、コンピュータが後手
BOARD_EMPTY 先手も後手もコンピュータ
その他（例：None） 先手も後手も人間（人間対人間）

yomi_depth はコンピュータの先読み深さである。 人間対人間のときは使われない（None でよい）。

def othello(sente_gote, yomi_depth=None):
    board = initial_board()
    turn = BOARD_SENTE   # 初期手番は常に先手
    count = 1
    ...

処理の流れは次の通りである。

  1. state = board_state(board, turn) で現在の状態を確認する。
  2. state が float なら終局なので game_result(board, state) を呼んで終了する。
  3. state == "pass" なら、パスのメッセージを表示し、手数を増やさずに手番だけ交代する。
  4. それ以外なら、盤面を表示し、コンピュータ担当なら play_by_machine(board, turn, count, yomi_depth)、人間担当なら play_by_human(board, turn, count) を呼ぶ。
  5. 手番を交代し、手数を進める。

play_by_machine は次章で実装するので、ここでは未定義のままにし、sente_gote が「コンピュータ担当を含まない値」（例えば None）のときだけ動かせるようにしておく。

Important重要：初期手番は常に先手

turn の初期値は BOARD_SENTE である。 sente_gote の値ではない点に注意せよ（sente_gote はコンピュータの担当を表すだけ）。
